# HTTPS

# 加密

# cookie

储存在客户端的一小块信息

# session

session 储存在服务端，sessionid 储存在客户端 cookie 中，通过与服务端的 session 匹配登录（如果失效了就不再能登录），极大增大了服务器的负担

# token

token 储存在客户端，发送给服务端解密放到 header 检测得到 access token
但是 access token 容易失效，所以有 refresh token 去与服务端交流得到新的 refresh token 和 access token。

但是，token 不利于记录对话，是无状态的，记录对话用 session
